# Skill: pm-risk-detect
# スプリント中リスク検出・早期アラート

## 概要
進行中のスプリントで遅延・品質・ブロッカーのリスクを自動検出し、
PMに早期アラートを送る。怠惰・無能メンバーの問題を早期発見するのに特に有効。

## 使い方
```
/pm-risk-detect --sprint "Sprint 1"
/pm-risk-detect --sprint "Sprint 1" --member charlie
```

## プロンプト

```
あなたはAI PMアシスタントです。現在のスプリント状況を分析してリスクを検出してください。

## スプリント: {sprint_name}
残り日数: {days_remaining}日
現在の完了率: {completion_rate:.0f}%

## チケット一覧
{ticket_list}
（各チケット: キー / タイトル / 担当者 / ステータス / 優先度 / 作成からの経過日数）

## メンバー特性（リスク評価に使用）
- チャーリー（dev-a）: 怠惰傾向。スプリント終盤に突然ブロックを報告する傾向あり
- ダナ（dev-a）: 品質問題。PRが何度も差し戻される傾向あり
- グレース（dev-b）: レビュー放置。他メンバーのブロッカーになる傾向あり
- ケン（dev-c）: 作業ミス。設定変更後の確認を怠る傾向あり

## 検出してほしいリスク
1. **遅延リスク**: 残り日数に対してIn-Progressが多すぎる / 進捗が遅いメンバー
2. **品質リスク**: 同一担当者のチケットが繰り返しDoneから戻ってくる
3. **ブロッカーリスク**: blockedのチケットが増加 / レビュー待ちが長期化
4. **コミュニケーションリスク**: 特定メンバーのチケットが長時間更新なし

## 出力形式
**🚨 リスク検出レポート: {sprint_name} (残り{days_remaining}日)**

### 🔴 高リスク（即アクション必要）
{高リスク項目のリスト。なければ「なし」}

### 🟡 中リスク（経過観察）
{中リスク項目のリスト。なければ「なし」}

### 💡 推奨アクション
{優先順位付きの具体的なアクションリスト}

### 📊 メンバー別進捗
| メンバー | Done | In Progress | Todo | コメント |
```

## 活用場面
- デイリー実行（朝会前）: チームの状況をPMが把握
- スプリント中盤（7日目）: 残り期間でのスコープ調整判断
- ブロッカー発生時: 即時分析・対応方針決定

## 発火タイミング（自動化する場合）
- 毎朝9時（cron: `0 9 * * 1-5`）
- blockedラベルが付いた時（Jira Automation Webhook）
- スプリント残り3日になった時

## 会社横展開時のカスタマイズポイント
- `メンバー特性` セクションを実際のチームに差し替える
- リスク閾値（完了率・残り日数）をプロジェクト規模に合わせて調整
- 通知先（Discord/Slack/メール）を環境に合わせて変更
